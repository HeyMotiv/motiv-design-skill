Collab Theme Reference

15 accent colors and 11 fonts a creator can choose from. This document shows exactly how each accent color is applied across the follower-facing collab details page.

How accent colors are applied

When a creator picks an accent color (e.g. Violet), it generates a 4-tier scale. Each tier is used for specific UI elements on the follower's collab details page.

The 4-tier scale (matches Figma Secondary/<color>/<tier>)

Tier Description Used for
/100 Strongest, darkest tone of the accent Themed text — large stat numbers ($83.67, 2/6 days, 800/2,100 kJ), inline links (More, Sync, Why these?), themed icons
/60 Medium tone — mid-saturation Filled progress bar segments, top of the photo overlay gradient, active progress dot fill
/30 Light tint — pastel Pill backgrounds (e.g. "Unlock MVP Bonuses"), soft accent fills, mid-stop of the photo overlay gradient
/0 Lightest tint — near-white wash Page background, empty progress bar segments, bottom of the photo overlay (so the image fades seamlessly into the page)

Photo overlay gradient (CSS)

A vertical gradient applied on top of the hero photo. It transitions from the solid /60 tone at the top, fades through transparent stops, and lands on the page background /0 at the bottom — creating the illusion that the photo melts into the page.

background: linear-gradient(180deg, /60 0%, /60 @ 20% alpha 26%, /60 @ 60% alpha 48%, /30 @ 80% alpha 66%, /0 92% );

Element-by-element mapping

  • /100Big stat numbers, "$" amounts, themed inline links, themed icons
  • /60Filled progress bar segments, gradient top stops, active progress-week dot ring
  • /30Filled pill background (e.g. "Unlock MVP Bonuses"); border for outlined pills (e.g. "$40 Hold"); gradient mid-stop
  • /0Page background, empty progress segments, outlined pill background (e.g. "$40 Hold"), gradient bottom stop

Two pill styles — different tier mappings

Pill style Background Border Text / icon
Filled tag — e.g. "Unlock MVP Bonuses" /30 none /100
Outlined pill — e.g. "$40 Hold" /0 /30 (1px) /100

Static elements — never themed

These elements use the same hard-coded colors regardless of which accent the creator picked. Don't plug accent variables into these.

Element Property Value
Profile tag on hero card (e.g. "@sarah_rides") Background white @ 40% opacity (Elevation/Medium)
Profile tag text Text Primary/100 — #2B2E34
Hero card title (e.g. "Dog Lovers Who Cycle") Text Surface/White — #FFFFFF
Hero card date range (e.g. "Jan 5 – Feb 15") Text Surface/White — #FFFFFF
"THIS WEEK" / section labels over the photo Text Surface/White — #FFFFFF
Stat-card backgrounds (the white panels) Background Surface/White — #FFFFFF
Secondary / unit text (e.g. "/6 days", "1,300 kJ left", "4 days to go", "Wk 1") Text Primary/60 — #6B7280
Section labels in the body (e.g. "YOUR PERKS", "STATS", "DON'T MISS") Text Primary/60 — #6B7280
Dividers / borders inside white cards Border Surface/Border — #E5E7EB
Body text (general) Text Primary/100 — #2B2E34

Why these stay static: elements layered over the photo gradient (titles, profile tag, "THIS WEEK") need consistent legibility across all 15 accents and any photo. Body-level greys and white cards keep the design feeling like Motiv, not just "a colored screen."

Implementation notes

Token naming. In Figma the variables are named Secondary/<Color>/<Tier> (e.g. Secondary/Purple/100). Suggested CSS-variable mirror:

--accent-0:   /* lightest tint   — page background, empty bars       */
--accent-30:  /* light tint      — pills, soft fills                 */
--accent-60:  /* medium          — filled bars, gradient top stop    */
--accent-100: /* darkest         — themed text, links, icons         */

Reference example from Figma (Purple). These are the exact values the design system currently ships:

  • Secondary/Purple/100 = #6B4E71
  • Secondary/Purple/60 = #775C7C
  • Secondary/Purple/30 = #C9B1CE
  • Secondary/Purple/0 = #F7F0FA

Photo overlay. The gradient is the same shape for every accent — only the color varies. The overlay's bottom stop must match the page background /0 exactly so the image "fades" into the page seamlessly.

Fonts. Load only the weights actually used (400 + 700). The chosen font is applied only to the collab title (hero card + detail page header). Body text always stays in Roboto for system consistency.
