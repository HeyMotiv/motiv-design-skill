## Getting Started

### 1. Review the Prototype

Open `prototype/index.html` in your browser to see the interactive prototype. This gives you a feel for the app before diving into specs.

### 2. Import Design Tokens

**For CSS/Web:**
```css
@import 'tokens/tokens.css';

/* Then use variables */
.button {
  background: var(--color-primary);
  padding: var(--spacing-md);
}
```

**For JavaScript/React:**
```javascript
import tokens from './tokens/tokens.json';

const styles = {
  backgroundColor: tokens.colors.primary.value,
  padding: tokens.spacing.md.value
};
```

**For Tailwind:**
```javascript
// tailwind.config.js
const designTokens = require('./tokens/tailwind.config.js');
module.exports = {
  ...designTokens,
  // your other config
};
```

- **GREEN-LINES.md**: Accessibility requirements (must-haves)
- **RED-LINES.md**: Detailed specs per screen

## Key Design Decisions

### Spacing System

Base unit: **{{SPACING_BASE}}**

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-xs` | {{XS}} | Tight spacing (icons, inline elements) |
| `--spacing-sm` | {{SM}} | Small gaps (between related items) |
| `--spacing-md` | {{MD}} | Default spacing (padding, margins) |
| `--spacing-lg` | {{LG}} | Section spacing |
| `--spacing-xl` | {{XL}} | Large gaps (between major sections) |

### Color Usage

| Token | Usage |
|-------|-------|
| `--color-primary` | CTAs, active states, links |
| `--color-secondary` | Secondary actions, accents |
| `--color-surface` | Card backgrounds, input backgrounds |
| `--color-background` | Page background |
| `--color-text-primary` | Headings, important text |
| `--color-text-secondary` | Body text, descriptions |

### Typography Scale

| Token | Size | Usage |
|-------|------|-------|
| `--font-size-h1` | {{H1}} | Page titles |

## States & Loading

Every screen should handle:

1. **Default** — Normal state with data
2. **Loading** — Skeleton or spinner while fetching
3. **Empty** — No data yet, encourage action
4. **Error** — Something went wrong, offer retry

See RED-LINES.md for screen-specific state designs.

## Accessibility Checklist

Before shipping, verify:

- [ ] All text meets contrast requirements (4.5:1 for body, 3:1 for large)
- [ ] All interactive elements have 44×44pt minimum touch targets
- [ ] Focus indicators are visible on all interactive elements
- [ ] Screen reader labels are present on all images and icons
- [ ] Keyboard navigation works through entire flows
- [ ] Animations respect `prefers-reduced-motion`

See GREEN-LINES.md for full accessibility requirements.
